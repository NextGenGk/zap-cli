# zap

> Push code. Just type `zap`.

`zap` replaces your entire git push workflow with a single command. It stages
your changes, suggests (or AI-generates) a commit message, commits, and pushes
— with sensible guardrails (main-branch warnings, pre-push checks, dry runs,
and one-step undo).

## Installation

```bash
npm install -g zap-git
zap init
```

`zap init` runs automatically the first time you call `zap` in a repo with no
local zap configuration.

## Quick start

```bash
cd your-project
zap          # stage, commit (with a suggested message), and push
```

```

  M  src/auth/Login.jsx

  D  src/auth/old.jsx

     → github.com/yourname/repo/commit/a3f91c2
```

## Commands

### `zap`

The default flow:

1. Verifies you're in a git repo and that there are uncommitted changes.
2. Lists changed files (`M`/`A`/`D`/`R`).
3. Warns before pushing directly to `main`/`master` (configurable).
4. Suggests a commit message from the changed files (or via `--ai`).
5. Stages everything, commits, and pushes the current branch (`-u origin <branch>`).
6. Prints the commit hash, push duration, and a link to the commit on GitHub.
7. Logs the push locally and, if connected, to your zap dashboard — non-blocking.

### `zap --ai`

Same flow, but the commit message is generated by NVIDIA NIM (`google/gemma-4-31b-it`)
from your `git diff`. You can accept, edit, or regenerate the suggestion. If the
API key is missing, the request times out, or the API errors, zap falls back to
the file-name based suggestion automatically — AI never blocks a push.

### `zap --undo`

Runs `git reset --soft HEAD~1`: undoes the last commit but keeps your changes
staged. Shows what was in that commit and asks for confirmation first.

### `zap --dry-run`

Simulates the entire flow — file list, branch warning, commit message, checks —
without staging, committing, pushing, or logging anything. Every line is
prefixed with `[DRY RUN]`.

### `zap --check` / `--skip-check`

`--check` runs your project's `lint`, `test`, and/or `build` npm scripts (in
that order, stopping at the first failure) before staging. If a check fails,
the push is blocked. `--skip-check` skips checks for this run even if your
config normally runs them.

Set the default behavior with:

```bash
zap config --check always   # always run checks before pushing
zap config --check never    # never run checks automatically
zap config --check ask      # ask each time (default)
```

### `zap log`

Shows your last 20 pushes (reverse chronological). If connected to a
dashboard, this fetches from there; otherwise it reads the local log.

```
2h ago   feat/login    feat(auth): add OTP verification    a3f91c2
1d ago   main          chore: update dependencies          b8e22d1
3d ago   fix/navbar    fix(nav): broken link on mobile     c91a3f4
```

### `zap init`

First-time setup wizard:

1. Checks/sets your git identity (`user.name` / `user.email`).
2. Checks/adds a GitHub remote.
3. Generates a `.gitignore` for your detected project type if missing.
4. Optionally connects to the zap dashboard (paste an API key).
5. Optionally enables AI commit messages (paste an NVIDIA NIM API key).
6. Shows a summary and offers to run a test push.

### `zap config`

```bash
zap config --show                   # show all current config
zap config --check always|never|ask # pre-push check behavior
zap config --ai on|off              # default AI mode
zap config --warn-main on|off       # main branch warning
zap config --reset                  # reset all config
```

### `.gitignore` generation

On `zap init` (and automatically on your first `zap` push if `.gitignore` is
missing), zap detects your project type — Node, Next.js, React, Python,
Django, Laravel, Go, Rust, or Java — and writes a merged `.gitignore` from
bundled templates, including project-specific extras (e.g. `.env.local` for
Next.js).

## Error handling

Every failure mode prints a clear message and (where applicable) a fix:

```
✗  Not a git repository
   Run git init first, then zap init to connect.

✗  Nothing to commit
   No changes detected in this directory.

✗  No remote configured
   Run: git remote add origin https://github.com/you/repo.git

✗  Push rejected (remote has changes)
   Pull first: git pull origin feat/login

✗  AI commit failed (API error)
   Falling back to manual message prompt.

✗  lint check failed
   Fix errors or run: zap --skip-check to push anyway.
```

## Environment variables

```
NVIDIA_API_KEY=       # AI commit messages (NVIDIA NIM — Gemma 4 31B)
ZAP_API_KEY=          # connects the CLI to your zap dashboard account
ZAP_SUPABASE_URL=     # base URL of your zap dashboard (set by `zap init`)
```

Environment variables always take precedence over values stored by `zap init`/`zap config`.

## Reliability

- Every git operation is wrapped with human-readable errors.
- Dashboard sync and AI calls are non-blocking with short timeouts — if
  they're unreachable, zap still completes the push.
- All core features (push, undo, log, config, dry-run, checks, `.gitignore`
  generation) work fully offline.
